# Warmaster Companion

Warmaster Companion is a mobile-first companion app for playing games of Warmaster Revolution. It helps players create and manage army lists, track battle results, calculate victory points, reference in-game charts, and keep a record of their miniature collections.

The app is built as an offline-first React Native application with local army data bundled in the repository. User-created armies, settings, battle state, and collection progress are stored locally on the device.

## Features

- Army builder for Warmaster Revolution factions.
- Dynamic and fixed points-limit support.
- Army validation for points, unit limits, upgrades, magic items, and Regiments of Renown rules.
- Battle tracker for combat resolution and victory point tracking.
- Lookup charts for scouting and blunders.
- Collection tracker for wishlist, owned, assembled, painted, and completed units.
- English and Spanish i18n support.
- Expo/EAS configuration for iOS, Android, and web development.

## Tech Stack

- React 19
- React Native 0.83
- Expo SDK 55
- TypeScript
- React Navigation
- React Context for app state
- AsyncStorage for local persistence
- Gorhom Bottom Sheet
- React Native Reanimated and Gesture Handler
- i18next / react-i18next
- Jest with jest-expo
- EAS Build and Expo Updates

The app currently runs from local JSON data under `src/data/json/wmr`. Generated C# schema files under `generated/csharp` are supporting artifacts and are not part of the mobile runtime.

## Project Structure

```text
App.tsx                         App entrypoint and provider shell
app.json                        Expo app configuration
src/context                     Shared state providers and builder logic
src/Navigation                  Screen stacks and feature screens
src/components                  Reusable UI components
src/data/json/wmr               Warmaster army, magic item, and rules data
src/i18n                        Translation resources
src/utils                       Data helpers, constants, formatting, assets
assets                          Images, icons, splash assets, fonts
generated/csharp                Generated schema/modeling artifacts
android                         Native Android project generated by Expo
```

## Prerequisites

Install:

- Node.js
- Yarn or npm
- Expo CLI through `npx expo`
- Android Studio for Android emulator/device development
- Xcode for iOS development on macOS

For native builds, you will also need an Expo/EAS account and the EAS CLI.

## Getting Started

Install dependencies:

```bash
yarn install
```

Or, if you prefer npm:

```bash
npm install
```

Start the Expo dev server:

```bash
yarn start
```

Or:

```bash
npm start
```

From the Expo dev server, open the app in an emulator, simulator, physical device, or web browser.

## Army Data

The current army data version is configured in `app.json` under:

```json
"extra": {
  "armyVersion": 2.262
}
```

Army and rules data lives in `src/data/json/wmr`, with current v2 data grouped through `src/data/json/wmr/v2/_armies.ts`.

When changing army data or builder logic, be careful with saved user data. Existing users may have older army lists stored locally, so changes should preserve the existing `AsyncStorage` keys and saved army-list shape unless a migration path is added.

## Notes for Contributors

- Keep builder calculations and validation behavior backward-compatible with existing saved armies.
- Prefer focused tests around pure builder logic when changing points, limits, or validation rules.
- Avoid changing local persistence keys without a migration strategy.
- Use the existing navigation, context, and component patterns before adding new app-wide architecture.
